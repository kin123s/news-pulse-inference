# 📐 프로젝트 전역 규칙 (Project-wide Rules)

> 본 문서는 회사 내 표준 AI 협업 환경(`.ai` 디렉토리)을 위한 전역 규칙입니다.
> 각 프로젝트 하위 워크스페이스의 `.ai/rules.md`는 이 규칙을 상속하며, 추가 규칙을 정의할 수 있습니다.

## 1. 패키지 관리

- **패키지 매니저**: 프로젝트 표준에 따르되, Node.js 환경의 경우 가급적 `pnpm`을 권장합니다.
- **lockfile**: 사용 중인 패키지 매니저의 lockfile(예: `pnpm-lock.yaml`, `package-lock.json`)은 반드시 커밋합니다.

## 2. 경로 및 스토리지 정책

- **OS 환경 독립성**: 본 프로젝트는 Windows, Mac, Linux 등 다양한 환경에서 개발될 수 있으므로, 모든 스크립트와 설정 파일 내의 경로는 **상대 경로(Relative Path)** 또는 **OS 독립적인 환경 변수**를 사용해야 합니다. 절대 특정 드라이브 문자(예: `C:/`, `D:/`)를 하드코딩하지 마십시오.
- **대용량 데이터 분리**: 1GB 이상의 아카이브, `.bin`, 백업 등 대용량 데이터는 로컬 프로젝트 폴더 외의 외부 스토리지나 NAS를 활용하도록 구성합니다.
- **호스트 직접 설치 지양**: 각종 패키지, 언어(Node.js, Python 등)는 호스트 OS에 직접 설치하는 것을 지양하고, 가급적 **Docker** 또는 **Dev Container** 환경을 구성하여 사용합니다.

## 3. 보안 (Zero-Trust)

- API 토큰, 시크릿 키 등 민감한 정보는 절대 코드나 설정 파일에 하드코딩하지 않습니다.
- `.env` 파일 등 시크릿 정보를 담은 파일은 반드시 `.gitignore`에 포함되어야 합니다.
- 외부 포트 노출 시 인증/인가 절차를 필수로 적용해야 합니다.

## 4. 디렉토리 컨벤션

- 프로젝트 루트 및 주요 서브 워크스페이스에 `.ai/rules.md`를 필수 생성/유지합니다.
- 반복되는 빌드, 배포, 테스트 등의 작업은 `.ai/workflows/` 디렉토리에 마크다운(`.md`) 형태의 워크플로우로 문서화해야 합니다.
- 특정 역할을 수행하는 에이전트의 프롬프트나 지침은 `.ai/roles/`에 배치합니다.

## 5. 코딩 표준

- 회사의 코딩 컨벤션을 우선적으로 따릅니다.
- linter(예: ESLint)와 formatter(예: Prettier) 설정을 준수하여 일관성 있는 코드 품질을 유지합니다.

## 6. 에이전트 작업 원칙

- **세션 상태 관리 필수**: AI 에이전트는 새로운 대화나 작업을 시작할 때 무조건 `.ai/state/sessions/` 디렉토리 내의 최신 세션 파일(`session_*.md`)들을 먼저 읽고 진척 상황과 문맥(Context)을 완벽히 숙지해야 합니다.
- 복잡한 작업이나 다단계 작업 진행 시, 에이전트는 작업 내용과 다음 단계를 `.ai/state/sessions/`에 파일로 강제 기록 및 업데이트하여 핸드오프(Handoff)가 가능하게 해야 합니다.
- 작업 완료 시, 다음 담당자(또는 다른 AI 에이전트)를 위해 **작업 인수인계서(Handover Report)** 양식에 맞춰 문서를 남겨야 합니다.

## 7. Git 브랜치 및 커밋 원칙

- **`main` 브랜치 직접 Push 절대 금지**: 모든 작업은 반드시 기능/이슈별 브랜치(예: `feat/issue-number`, `fix/bug-name`)를 생성하여 진행하십시오.
- **Pull Request (PR) 필수**: 코드 변경 사항은 항상 PR을 통해 원본 브랜치에 병합(Merge)되어야 하며, 코드 리뷰 절차를 거치는 것을 원칙으로 합니다.
- **Commit 컨벤션**: Conventional Commits 형식을 준수하십시오 (예: `feat: Add session context`, `fix: Resolve PR workflow bug`).
